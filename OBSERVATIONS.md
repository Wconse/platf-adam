## ⚠️ РЕТРАКЦИЯ (раунд-9 ревью коллеги) — ЧИТАТЬ ПЕРВЫМ

«Phase 0 закрыта» и «результат SP подтверждён» — **ОТОЗВАНЫ**. Ниже по тексту
остаются пометки [ПОДТВЕРЖДЕНО] на SP-выводах — они относятся к КОРРЕКТНОСТИ
наблюдения («E→E не распространяется в изолированной SP-микросхеме при референсных
параметрах»), но НЕ к его интерпретации и НЕ к закрытию фазы. Три причины:

1. **E_GABA: ДВА ПУТИ ДАННЫХ. Моя SP-работа взяла НЕВЕРНЫЙ. (КОРРЕКЦИЯ round-9.5)**
   Движок читает GABA-реверсал из `ta_egaba_{region}.npy` (kernel: E_GABA_now =
   d_ta_egaba[step]; статический `ap.E_GABA` из конфига ЗАГРУЖАЕТСЯ, но НЕ
   используется — main_multiregion.cu:605 vs 290/313). Есть ДВА источника этих файлов:
   - **Путь A — dev_state (ПРАВИЛЬНЫЙ):** `phase0_tools.py:180-190` строит
     DevStateVector, `dev_state.py:175-188` даёт egaba_mV по сигмоиде (E_early=−45,
     center_dw SP=26.5) → на dw-20 = **−45 мВ (деполяризующий)** для всех регионов.
     `engine_runner.py:156-159` пишет ta_egaba из этого. → ФАКТОРИАЛ rounds 4–8
     РЕАЛЬНО шёл при −45 мВ. Их «verified fact: E_GABA −45» был ВЕРЕН.
   - **Путь B — сырой bundle (УСТАРЕВШИЙ хардкод):** `reference/multiregion/ta_egaba_*.npy`
     все = −75 (записаны `gen_multiregion.py:61` «simplified constant for M3»).
   **Моя SP-работа грузила сырой bundle напрямую (путь B) → шла при −75, а не −45.**
   Значит мой тест «E_GABA −45 vs −75 → нет разницы» сравнивал −45 с −75 в SP, НО
   базовая рабочая точка SP-микросхемы у меня была −75 (неверная для dw-20). Это НЕ
   дефект генератора кода (dev_state корректен) — это МОЙ промах в выборе источника
   данных для зеркала.

   **ПЕРЕМЕРЕНО при −45 (round-9.5):** вывод «E→E не распространяется» УСТОЯЛ.
   e_desc=0 при E_GABA −75/−60/−45 (seed 25%, baseline молчит, spont_late=0).
   Прямая запись мембраны не-seed E нейрона во время каскада: пик v = −54.84 мВ при
   −45 vs −54.93 мВ при −75 (VT=−50). Сдвиг знака GABA ВКЛЮЧАЕТСЯ (пик на 0.09мВ
   выше), но эффект ничтожен: g_gaba на не-seed E = 0.038 нС — каскад это ОДНО
   feedforward-поколение, а деполяризующей GABA-петле нужно НЕСКОЛЬКО поколений
   чтобы накопиться. → наблюдение «E→E мертва» НЕ артефакт неверного −75; держится
   и при корректном −45. Проксимальная причина та же: одиночное E→E поколение
   подпорогово независимо от полярности GABA (разреженность S_SP_EE + шунт).

2. **SP не подключён к иерархии — тоже дефект, не дизайн.**
   Иерархия (`gen_multiregion.py:324`, `gen_multiregion_scale.py:371`): TH→P1→P2→A1→A2,
   МИНУЯ SP. Спецификация (`Present.md:22,42`): «Brainstem→Thalamus→Subplate», SP =
   «transient scaffold layer». Заявленная роль посредника TH→SP→P1 в топологию НЕ
   заведена. → даже если SP забёрстит, P1 этого не увидит. Анализ «почему SP не
   бёрстит» = анализ изолированного острова без био-функции в системе.

3. **СОБСТВЕННАЯ ОШИБКА: я дважды написал «SP receives no external input» — НЕВЕРНО.**
   В `aff_config.json` есть `S_GSW_SP: GSW→SP_E` (1024 синапса, src_N=32),
   `aff_rate_GSW = 1.0 Гц` постоянно. SP получает сенсорный GSW-афферент. Я смотрел
   только `projections` в network_config, не проверил `aff_config`. → numpy-зеркало
   (без афферентов) валидно ТОЛЬКО для afferent-silent окна; это НЕ проконтролировано
   явно в Q3b. Весь sustained-анализ игнорировал реальный GSW-вход в SP.

4. **Q5 протокола пройден нечестно.** «SP должна бёрстить» — направление, не
   количественный таргет (частоты/длительность бёрстов/IBI). Фаза закрыта при
   непройденном стоп-вопросе. Отозвано.

**Реальный статус: Phase 0 НЕ закрыта.** Наблюдение «E→E мертва» реально и УСТОЯЛО
при корректном E_GABA=−45 (round-9.5, см. п.1). Но интерпретация «SP не выполняет
био-функцию» по-прежнему не установлена из-за п.2 (нет TH→SP→P1) и п.3 (GSW не учтён
в зеркале).

### КОРРЕКЦИЯ round-9.5 к самой ретракции (после разрешения противоречия E_GABA):
Эта ретракция изначально ПЕРЕ-обобщила, намекая что дефект −75 заразил всё. Проверка
путей данных показала обратное: ФАКТОРИАЛ rounds 4–8 шёл через dev_state (путь A) при

и даже там вывод про E→E устоял при перемере на −45. Итого из 4 пунктов ретракции:
- п.1 (E_GABA): переквалифицирован — мой промах источника данных, НЕ дефект кода;
  наблюдение перемерено и устояло. Факториал не затронут.
- п.2 (нет TH→SP→P1): СТОИТ — реальный пробел топологии во всех путях (dev_state
  топологию не трогает, только веса).
- п.3 (GSW афферент не в зеркале): СТОИТ — моя ошибка, зеркало валидно лишь в
  afferent-silent окне.
- п.4 (Q5 нечестно пройден): СТОИТ — фаза была закрыта преждевременно.
«Phase 0 закрыта» остаётся ОТОЗВАННЫМ, но по причинам п.2/п.4 (неполнота Phase-0

### [ВЫБРОШЕН] MR-эстиматор (Wilting–Priesemann) на driven-трейсах
- Прошёл контроль на синтетике с БЕЛОЙ иммиграцией.
- НО на реальных driven-трейсах m_hat (0.978/0.955/0.912 при bin 0.5/1/2мс) почти
  точно равен автокорреляции OU-шума exp(-bin/20мс) = 0.975/0.951/0.905.
- Вывод: на наших данных MR читает 20мс-таймскейл цветного OU-входа, НЕ нейронное
  ветвление. Иммиграция у нас цветная, а метод предполагает белую. Q3 поймал.

### [ВЫБРОШЕН] Perturbation через carry_state движка
- Давал 0 спайков даже у принудительного seed.
- Причина (важно!): **maturation gate движка**. SP_E имеет t_bir=0, T_mat=800мс.
  На шаге 0: mat = clamp((t - t_bir - 0.5dt)/T_mat, 0, 1) = 0, и кернел выполняет
  `v = V_reset; return` ДО проверки спайка → любой seed-вольтаж затирается к -65мВ.
- Это не баг, это так движок и устроен. Подход через движок для перчтурбации непригоден.

## [ПОДТВЕРЖДЕНО] ge0 — это nS, не артефакт единиц
Диагностика токов одного SP_E при покое (ge_scale=1, mat=1, sig=0, v≈-55мВ):
- I_ge = ge*(E_e - v) = 10*(0-(-55.4)) = +554 пА (деполяризующий)
- I_gi = gi*(E_i - v) = 18*(-80-(-55.4)) = -443 пА (гиперполяр.)
- I_leak = -gL*(v-EL) = -146 пА; I_bi = +80 пА; сумма ≈ 0 при равновесии.
Вывод: 0 Гц в детерминированном покое — НЕ баг единиц. gi0=18 (E_i=-80) уравновешивает
ge0=10, фиксируя покой на ~-54..-55мВ, чуть ниже VT=-50. Без флуктуаций (sig=0)
порог не пересекается → молчание. Это корректное равновесие, согласуется с truebare=0.

## [ПОДТВЕРЖДЕНО] SP не может само-распространять возбуждение (E→E мёртв)

ОБНОВЛЕНИЕ: ранняя формулировка «m≈0.33, субкритично» была ОШИБОЧНОЙ интерпретацией
(Q1/Q7 поймал). m считалось по СУММАРНЫМ спайкам сети, а pop-resolved разбивка
показала: **e_desc (возбуждающие потомки) = 0 ВО ВСЕХ строках**. Все «потомки» —
это PV-интернейроны, срабатывающие один раз. То есть я мерил feedforward E→PV
рекрутирование, а НЕ рекуррентное E→E ветвление. Тавтология устранена.

### Что измерено корректно (валидированное зеркало, mat=1, sig=0, baseline молчит):
- Синхронный залп до 200 SP_E нейронов (25%) → **0 возбуждающих потомков**, только
  85 PV-спайков (одно поколение feedforward). Каскад гаснет за ~6–7мс (1 поколение).
- Стабильность (Q4): e_desc=0 при любом seed_frac (0.02..1.0) и любом валидном
  ge_scale (1.0..1.25, где baseline ещё молчит). m нечувствительно к settle/window ±20%.
- kill_inh (обнулить ВСЕ тормозные веса): e_desc по-прежнему 0. → НЕ торможение душит
  каскад. (Тормозная обратная связь даже не успевает включиться — каскад умирает за
  1 поколение.)
- E→E scaling ×1..×12 (только S_SP_EE): e_desc по-прежнему 0; мембрана не-seed
  E-нейрона достигает лишь -53.7мВ (при ×12) vs -55.3мВ (×1).

### Механизм [ПОДТВЕРЖДЕНО прямой записью v]:
1. Одиночный E→E залп ПОДпорогов для AdEx-нейрона: мембрана не-seed E достигает
   только ~-54мВ, не дотягивая до VT=-50 (мягкий порог запуска экспоненты), тем более
   до V_spike=-30 (жёсткий cutoff). Экспоненциальный апстрок не зажигается → нет спайка.
2. PV (SP_I) имеет V_spike=-50 НАПРЯМУЮ (LIF), поэтому тот же вход пересекает его
   порог → PV спайкует. Отсюда асимметрия: залп subthreshold для E, suprathreshold для PV.
3. Шунтирование: тонический фон даёт ge0=10 + gi0=18 = ~28нС постоянной проводимости
   против gL=10нС. Мембрана сильно зашунтирована → одиночный транзиентный E→E импульс
   двигает её лишь на ~1.6мВ.

### [ОТКЛОНЁН/НЕВАЛИДЕН] kill_gi0_bg (обнуление фоновой gi0)
Давал e_desc=-200 (сам seed не регистрируется), v=-65. Обнуление фоновой тормозной
проводимости дестабилизирует фазу settle. Вывод по этому пробнику НЕ делается.

### ВЫВОД: При референсной рабочей точке E→E рекуррентность SP на порядок слабее,
чем нужно для само-поддерживающегося каскада. Проксимальная причина — одиночный
синхронный залп подпорогов для AdEx-апстрока (≈-54мВ vs VT -50), сильно зашунтирован
тоническим фоном. Это напрямую объясняет, почему SP не бёрстит сам.

### Что НЕ проверено (Q9, для след. шага):
- Устойчивый (а не одиночный) вход в SP_E — возможно бёрст требует sustained drive.
- Депол-GABA (E_GABA выше): при молчащем фоне GABA почти не открыта, эффект вероятно
  мал, но не измерено.
- Связь с реальным режимом dw20: при mat<0.5 (реальная незрелость) SP ещё слабее.
- Какой ee_scale ИЛИ снижение шунта вернёт E→E распространение (для Phase-1 калибровки).

## [ПОДТВЕРЖДЕНО] Sustained drive ТОЖЕ не вызывает бёрст (закрыт вопрос Q10 рецензента)

Тест (sustained_sp.py): драйв 30% SP_E постоянным током drive_ms=100мс, затем выкл,
наблюдение 300мс. Метрика e_nondriven = спайки НЕ-драйвируемых E во время драйва;
persistence_ms = как долго сеть активна после выключения драйва. Зеркало пере-провалидировано
после правки run() (persistent step counter): (A) v rms 1.6e-5мВ, (B) частоты ≤0.2%.

Результаты (референсные веса):
- I_drive 150..800пА (до 4× реобазы), 240 драйвируемых E: **e_nondriven = 0 ВЕЗДЕ**.
  Драйвируемые клетки спайкуют сильно, надёжно рекрутируют PV (87..126 спайков),
  но НИ ОДНОГО E→E рекрутирования.
- Реверберации нет: persistence после выкл = 2.6мс ≈ одна синапт. задержка. e_post=23
  при 500пА — это драйвируемые клетки доживают последний рефрактерный цикл, не
  само-поддержание (умирает за <3мс, << NMDA τ=100мс).
- E→E ×1..×8: e_nondriven по-прежнему 0. Даже ×8 веса не проталкивают не-драйв. E
  через порог (шунт ~28нС + PV-обратная связь зажимают).
- Депол-GABA E_GABA -75/-55/-45: e_nondriven=0 при всех. GABA-полярность НЕ спасает
  при этой активности; узкое место — E→E путь, не GABA. Подтверждает прежний
  факториал (E_GABA нерелевантна режиму) — теперь и под structured sustained drive.

### ОБЩИЙ ВЫВОД по SP [НАБЛЮДЕНИЕ подтверждено; ИНТЕРПРЕТАЦИЯ отозвана — см. РЕТРАКЦИЮ]:
НАБЛЮДЕНИЕ (корректно, оба теста — импульс и sustained): при референсных параметрах
bundle SP-микросхема НЕ само-поддерживает возбуждение — E→E рекуррентность
функционально мертва (0 E-потомков при импульсе и при sustained до 4× реобазы).
Проксимальные причины (прямой записью v):
(1) одиночный/постоянный E→E вход подпорогов для AdEx-апстрока (мембрана ≈-54мВ vs VT -50);

### Инструмент валидирован (Q3, контроль на известном ответе)
Зажимаем V, держим постоянную g_GABA, сверяем стационар симулятора с аналитической

γ выведен из первых принципов: γ = 1/(F·Vol). Vol=1пл → 0.0104 мМ/(пА·с).
(Kaila 1997 Prog Neurobiol; Staley & Proctor 1999 J Physiol.)

### Находка 1 — ЗНАК сдвига (crossover при V = E_Cl) [ПОДТВЕРЖДЕНО, безартефактный плато-тест]
Направление потока Cl⁻ во время burst'а определяется ТОЛЬКО положением Vm
относительно E_Cl(=−45 при dw20):
  Vm > −45 (плато GDP −40..−30) → Cl⁻ ВХОДИТ → cl растёт → E_GABA ДЕПОЛЯРИЗУЕТСЯ (loading)
  Vm < −45                       → Cl⁻ ВЫХОДИТ → E_GABA гиперполяризуется (efflux)
  Vm = −45                       → ноль (точка перехода)
Никакого "бесплатного" истощения нет — это чистый кроссовер из Нернста+driving force.

### Находка 2 — РОАДМАП ПЕРЕПУТАЛ ЗНАК для депол-GABA режима [СИЛЬНО ПОДДЕРЖАНО]
Макропауза роадмапа требует, чтобы burst делал E_GABA БОЛЕЕ гиперполяризующим
(efflux, падение к −60) → нужно Vm < −45 во время burst'а.
